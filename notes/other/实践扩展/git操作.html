# Git 基本操作：请每天背一遍，好吗？

1、首先将文件 git add 到暂存区 Index，告诉 git 那些文件需要被 git 所管理

2、然后通过 git commit 将暂存区的目录树同步到本地仓库中

3、最后同步到远端仓库中

# 提交操作

(1)将工作区的所有文件提交到暂存区中

git add .//仅提交被修改的和新增的文件，不提交删除的文件
git add -u //仅提交被修改的和被删除的文件，不提交新增的文件
git add -A //提交所有的变化，是上面两个命令的结合
1
2
3

(2)将暂存区的文件提交至本地仓库

git commit -m msg
1

msg 的常见类型：

  • feat: 新增的功能
  • fix: 修复的 bug
  • style: 修改的样式
  • refactor：重构（即不是新增功能，也不是修改 bug 的代码变动）
  • docs：文档（documentation）
  • perf: 优化相关，比如提升性能、体验
  • test：增加测试
  • chore：构建过程或辅助工具的变动
  • revert: 回滚到上一个版本

# 撤销操作

  • 场景 1：撤销工作区的修改

    git restore filename
    1

  • 场景 2：撤销暂存区的修改

    git restore --staged filename
    1

  • 场景 3： 撤销本地仓库中的修改

    git reset --hard HEAD //将三处内容都置为 reset 节点的内容
git reset --soft HEAD //保留工作区和暂存区的修改，并将节点撤销前后的差异恢复至暂存区中
git reset HEAD //会保留工作区的修改，将暂存区置为 reset 节点的内容，并将撤销前后的差异恢复至工作区中
//或
git revert HEAD
    1
    2
    3
    4
    5

    reset 和 revert 的区别: reset 是将需要撤销的 commit 之后的所有 commit 都删除 revert 则是用新的 commit 去抵消需要撤销的那次 commit 中的修改，其之后 commit 操作依旧会被保留

# 远端仓库

(1) 将本地仓库与远端仓库进行关联

git remote add origin 远端仓库地址 //起了个名字叫做origin，也是git默认的叫法
1

解除本地仓库与远端仓库的关联

git remote -v //列出来有哪些绑定
git remote rm name
1
2

(2) 推送到远端仓库对应分支

git push -u origin master //首次推送时加上-u，会将本地分支与远端master分支关联起来，之后再push的时候就不用再加-u了
1

# 合并分支

//将当前分支的节点移动到b分支的节点处

//或
git rebase dev

merge 和 rebase 的区别： 前者会将分支和 master 上的操作合并，并生成一个新的节点 后者则会将两个分支合并为一个分支，不会产生新的节点

# 暂存修改的代码

//保存当前分支上未commit的代码
git stash
git stash save "备注的内容"
//恢复最近一次暂存的代码
git stash apply
//恢复最近一次的并删除记录
git stash pop
//删除最近一次的记录
git stash drop
//删除所有的记录
git stash clear

//当有多条记录，想恢复指定记录时
//列出stash所有的记录
git stash list
git stash apply stash@{n}
