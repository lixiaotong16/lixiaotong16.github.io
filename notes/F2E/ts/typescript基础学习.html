# 8.类型推论

可以不写类型注解的情况：声明变量并立即初始化时/决定函数返回值时

# 9.类型断言

有时类型推论得到的属性是一个比较宽泛的属性，其只有一些公共的方法，无法访问更具体属性所特有的方法，此时可以使用类型断言指定更加具体的类型

通过 as 关键字指定更加具体的类型

const aLink = document.getElementById("link") as HTMLAnchorElement;

# 10.字面量类型

使用场景：用来表示一组明确的可选值列表，比使用宽泛类型更严谨，通常配合联合类型进行使用

changeDirection("up");

(4)继承

类继承的方式有两种：通过 extends 继承父类/通过 implements 实现接口

继承是类与类之间的关系

实现是类与接口之间的关系：类必须实现接口中的所有属性和方法，相当于为一个类提供了一些约束

//implements的使用方法
interface Singale {
    name: string;
    sing(): void;
}
class Person implements Singale {
    name: string;
    sing() {
        console.log("lalala");
    }
}

# 2.类型兼容性

ts 采用结构化类型系统，也叫鸭子类型

当两个对象具有相同的形状，则认为它们属于同一类型

(1)对象/类/接口之间的兼容性：

成员多的可以赋值给成员少的

(2)函数之间的兼容性比较复杂

# 3.交叉类型

类似于接口继承（extends），用于组合多个类型为一个类型（常用于对象类型）

interface Person {
    name: string;
}
interface Contact {
    phone: number;
}
type PC = Person & Contact;
1

5
6
7

通过&实现将两个类型中的所有属性和方法合并为一个类型

注意使用接口继承与交叉类型之间的区别：

对于同名属性之间，处理类型冲突的方式不同

接口继承时，属性或方法会发生冲突

交叉类型则不会产生冲突，因为可以用冲突中的任意一种

# 6.映射类型

基于旧类型创建新类型（对象类型）

type PropKeys = "x" | "y" | "z";
type Type2 = { [key in PropsKeys]: number };
//相当于type Type2 = {x:number; y:number; z:number}
1
2
3

注意，这个只能在 type 中使用，不能在接口中使用

除了可以通过联合类型来创建映射类型外，还可以通过一个对象类型来生成

type Props = {a:number. b:string, c:boolean}
type Type1 = {[key in keyof Props]:number}
//相当于type Type1 = {a:number; b:number;c:number }
