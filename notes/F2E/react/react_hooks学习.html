setXxx()的两种写法:

  • setXxx(newValue): 参数为非函数值, 直接指定新的状态值, 内部用其覆盖原来的状态值
  • setXxx(value => newValue): 参数为函数, 接收原本的状态值, 返回新的状态值, 内部用其覆盖原来的状态值

注意点：

（1）setCount()是异步的，也就是说在更新完状态后，无法立即获得最新的值，可以通过回调函数的形式获得

setCount((count) => {
    count++;
    console.log(count); //获取到了最新的值
    return count;
});
//或者
setCount((count) => count + 1);

（3）useEffect函数的第二个参数：如果 useEffect 内部的操作依赖于某个最新的状态，则填入依赖项，该函数会在依赖的状态更新的时候执行。如果不填，则任何一个状态更新后都会执行当前的useEffect，如果填[]，则只会初始执行一次。

# useEffect 请求数据

(1) 一般建议把不依赖 props 和 state 的函数提到你的组件外面，并且把那些仅被 effect 使用的函数放到 effect 里面
(2) 如果 effect 的依赖是个函数的话，一定要在函数外面包一个 useCallback。如果是 props 传进来的函数，props 传进来的函数可以访问到 props 和 state。把它的定义包裹进 useCallback Hook。这就确保了它不随渲染而改变，除非它自身的依赖发生了改变。

```js
const getFetchUrl = useCallback(props.fetchData, [query]);
```

(3) 如果你对 async/await 熟悉的话，你会知道，每个 async 函数都会默认返回一个隐式的 promise。但是，useEffect 不应该返回任何内容，因此不能直接在 useEffect 中使用 async 函数，因此，我们可以不直接调用 async 函数，而是像下面这样:

```js
useEffect(() => {
    const fetchData = async () => {
        const result = await axios(
            "http://localhost/api/v1/search?query=redux"
        );

        setData(result.data);
    };

    fetchData();
}, []);
```

(4) 有些依赖内部值没有变，但是每次渲染都会重新生成一个引用，使得 useEffect 总是重复调用，因为 useEffect 是浅比较
可以将依赖转为字符串，使用 JSON.stringfy(xxx)
除此之外，最好的还是使用 useDeepCompareEffect，它使用的是深比较策略，不过性能上有所牺牲

# 4.useRef()

当一个信息用于渲染时，把它保存在 state 中。 当一个信息只被事件处理程序需要，并且改变它不需要 re-render 时，使用 ref 可能更有效。

ref 和 state 的区别：

  • 修改 ref 不会触发重新 render，修改 state 会
  • ref 是一个纯对象，他可以通过直接设置 current 属性的方式修改，state 需要通过状态设置函数来修改
  • ref 不能在 re-render 之间读取或写入，这样会导致结果不可控

常用于以下几种情况：

  • 存储 timeout ID
  • 存储和操作 DOM 元素
  • 存储不需要计算 JSX 的其他对象
const inputRef = useRef();
<input ref={inputRef} />;
//使用时
inputRef.current.focus();

# 5.useMemo()和 useCallback()

(1) memo

父组件更新，若子组件没有变化的话，希望不更新，则在定义子组件的时候，使用memo()函数包一下，相当于PureComponent

const Foo = memo(() => {
    return;
});
1
2
3

(2) 当父组件向子组件传递函数时，由于函数定义在父组件中，父组件每次 render 都会生成一个新的函数，因此，即使函数内容没有变，但是引用发生了变化，还是会引起子组件的重新渲染，因此可以使用useMemo和useCallback

const listRender = useCallback(()=>{
    ....
},[])
<Foo listRender={listRender}/>
1
2
3
4

第二个参数为依赖项，由于函数可能会依赖于某个状态，只有当这个状态发生变化的时候，才会生成一个新的函数，第二个参数可以填[]或者填依赖项数组

useMemo与useCallback差不多，不同的是useMemo会根据函数生成一个固定的值，而useCallback生成的是一个固定的函数。

useMemo 默认情况下其只会对 props 做浅层对比，遇到层级比较深的复杂对象时，表示力不从心了。对于特定的业务场景，可能需要类似 shouldComponentUpdate 这样的 API，这时通过 memo 的第二个参数来实现：

function arePropsEqual(prevProps, nextProps) {
    // your code
    return prevProps === nextProps;
}

export default memo(Button, arePropsEqual);

# 6.useReducer()

//用法
const countReducer = (state, action) => {
    if (action.type === "add") {
        return state + 1;
    }
};
const [count, countDispatch] = useReducer(countReducer, 0);
countDispatch({ type: "add" });
